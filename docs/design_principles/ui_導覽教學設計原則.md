# UI 導覽教學設計原則

## 一、用途

本文件用來保存「系統操作教學 / 導覽模式」的設計原則，供未來在 `D:\IES_TEST` 內的其他專案重複使用。

適用場景：

- HTML UI 教學導覽
- Web 系統 onboarding
- 操作流程引導
- Demo 模式教學
- 需逐步介紹畫面功能的內部工具

## 二、核心目標

導覽模式的目標不是只「提示使用者」，而是要做到：

- 使用者知道現在在看哪個功能
- 被介紹的區塊保持清晰
- 說明泡泡保持清晰
- 背景存在感降低，但不干擾閱讀
- 導覽結束後不佔用正常 UI 空間

## 三、主要設計原則

### 1. 教學入口要簡潔

- 平常 UI 上只保留一個明確入口，例如右上角 `使用教學`
- 不要在常態畫面中放大型教學面板
- 教學功能應在需要時才展開

### 2. 教學模式不應長期佔據版面

- 教學說明應採用浮動式 popup / bubble
- 平常不應額外佔用主畫面空間
- 使用者退出教學後，畫面應回到乾淨的正常狀態

### 3. 背景可弱化，但重點區塊必須清晰

- 使用 overlay 暗化背景
- 不要對整頁直接做 blur，避免連目標區塊也變糊
- 被介紹的目標區塊要提升到 overlay 之上
- 目標區塊需要保持原始清晰度與可閱讀性

### 4. 說明 bubble 必須獨立於背景效果

- bubble 應使用獨立 fixed layer
- bubble 必須位於最高層之一
- 不可放在被 blur 或透明疊層影響的父層內
- bubble 需具備高對比文字、清楚背景、可辨識邊框與陰影

### 5. 每一步都要有清楚視覺焦點

- 同一時間只突出一個目標區塊
- 當前目標需有明確高亮效果
- 可使用：
  - 外框
  - glow
  - 陰影
  - z-index 提升

### 6. 畫面要自動移動到目標區塊

- 教學切換步驟時，畫面應自動 scroll 到目標位置
- 不要讓使用者自行找畫面焦點
- 導覽的價值之一就是降低搜尋成本

### 7. 說明內容要短而明確

- 每一步只講一個核心功能
- 說明文字應聚焦：
  - 這一區是什麼
  - 使用者現在應做什麼
  - 這一步完成後的目的

### 8. 教學流程要可退出、可前進、可回退

- 至少提供：
  - 退出
  - 上一步
  - 下一步
- 最後一步完成後可自動退出或提示完成

### 9. Bubble 位置要自動避開邊界

- 若目標區塊右側空間不足，bubble 應改顯示在左側
- 若底部空間不足，bubble 應自動上移
- 小螢幕時需保留後續切換成底部卡片模式的彈性

### 10. 教學模式要可模組化重用

- 教學資料應獨立成 steps 陣列或設定物件
- 每一步至少包含：
  - title
  - anchor
  - detail
- 視覺邏輯與內容資料應分離，方便跨專案重用

## 四、推薦技術做法

### Overlay 層

- 使用 `position: fixed`
- 覆蓋全頁
- 採半透明深色暗化
- 不直接對整頁內容套 blur

### 目標區塊高亮

- 動態加上例如 `tour-active-target` class
- 使用：
  - `position: relative`
  - 高 `z-index`
  - `box-shadow`
  - `isolation: isolate`

### Bubble 層

- 使用 `position: fixed`
- 與 overlay 分開
- 高 `z-index`
- 高不透明背景
- 保留箭頭樣式以指向目標

### 教學資料結構

建議格式：

```js
[
  {
    title: "Step 1：準備資料",
    anchor: "upload",
    detail: "說明文字"
  }
]
```

## 五、視覺原則

### 清晰度優先

- 重點不是炫技，而是清楚
- 教學中的目標區塊與 bubble 必須優先於背景效果

### 對比優先

- bubble 文字與背景對比要高
- 按鈕狀態要清楚
- 進度要可讀

### 一次只強調一件事

- 避免同時高亮多個區塊
- 避免同一個步驟塞太多資訊

## 六、互動原則

### 使用者不應迷路

- 每一步都應知道「現在在哪裡」
- 要有步驟進度，例如 `2 / 4`

### 使用者不應被干擾

- 導覽模式要降低背景存在感
- 但不要破壞主要版面結構

### 使用者不應被強迫

- 教學可隨時退出
- 正常工作流不依賴教學模式才能操作

## 七、未來擴充建議

後續若要升級，可往以下方向：

- 首次開啟頁面時詢問是否開始教學
- 為 bubble 增加箭頭精準定位
- 小螢幕自動切到底部卡片模式
- Spotlight 挖洞效果
- 記住使用者是否已看過教學
- 多語系教學內容

## 八、命名建議

若未來在其他專案重用，建議使用以下命名：

- `tour-overlay`
- `tour-card`
- `tour-active-target`
- `guideStartBtn`
- `tourNextBtn`
- `tourPrevBtn`
- `tourExitBtn`

## 九、套用原則

未來若在其他專案中要做教學模式，優先遵守以下順序：

1. 先保證目標區塊清晰
2. 再保證 bubble 清晰
3. 再處理背景弱化
4. 最後再做動畫或視覺美化

## 十、備註

本原則來自 `project_cqe_qnrhelper/ui/index.html` 的導覽教學設計實作，可作為後續 UI 導覽模式的預設準則。
